# 购物数据可视化大屏 — 项目实施方案

## 项目概述

构建一个 **1920×1080** 全屏数据可视化大屏，以北京市海淀区地图为核心，展示购物数据、商铺分布、购买人购物轨迹及各类统计图表。整体风格追求 **深色科技感 + 赛博朋克风**，强调动态效果与沉浸式体验。

---

## 技术选型

| 层面 | 方案 | 理由 |
|------|------|------|
| **构建工具** | Vite | 极速冷启动、热更新 |
| **核心语言** | 原生 JavaScript + HTML + CSS | 无框架依赖，轻量高效 |
| **图表引擎** | Apache ECharts 5.x | 强大的 Geo 地图、Lines 飞线、各类统计图表支持 |
| **地图数据** | 海淀区 GeoJSON（阿里 DataV.GeoAtlas） | 标准行政区划边界 |
| **字体** | Google Fonts — Orbitron（标题）+ Noto Sans SC（正文） | 科技感英文 + 清晰中文 |
| **动画** | CSS Animations + ECharts 内置动效 | 入场动画、数字滚动、呼吸灯等 |
| **适配方案** | CSS `transform: scale()` 等比缩放 | 适配不同物理分辨率大屏 |

## 功能模块详细设计

### 模块一：中央地图与购物轨迹（核心）

| 特性 | 实现 |
|------|------|
| **底图** | 海淀区 GeoJSON，深蓝/深青色填充，亮色边界线 |
| **商铺标注** | `effectScatter` 涟漪散点，鼠标悬浮显示详情 |
| **购物轨迹** | `lines` 系列飞线效果，带箭头拖尾动画 |
| **轨迹切换** | 选中不同购买人，动态更新飞线路径，标注购买顺序 |
| **自动轮播** | 无操作时自动轮播不同购买人的轨迹 |
| **商铺标签** | 地图上显示商铺名称，悬浮显示畅销品、当日客流等 |

**商铺坐标（Mock 数据 — 海淀区真实经纬度范围内）**：

### 模块二：商铺信息实时轮播面板（右上）

自动滚动展示各商铺实时信息卡片：

| 展示字段 | 说明 |
|---------|------|
| 商铺名称 | 大字标题 |
| 今日客流 | 实时人数（模拟递增动画） |
| 今日销售额 | 带数字滚动效果 |
| 畅销品 TOP3 | 商品名 + 销量 |
| 商铺评分 | 五星评分（4.2 ~ 4.9） |
| 营业时间 | 09:00 - 22:00 |

**交互**：点击地图商铺时，右侧面板自动切换到该商铺详情。

---

### 模块三：统计图表组（底部）

#### 3.1 24小时销售趋势 — 折线图
- X 轴：0:00 ~ 23:00
- Y 轴：销售额（元）
- 特效：渐变面积填充、数据点发光、平滑曲线

#### 3.2 各商铺销售额排行 — 柱状图
- 横向柱状图，8 个商铺
- 渐变色柱体，数值标签显示在柱体右侧
- 悬浮高亮交互

#### 3.3 客流量分析 — 雷达图
- 维度：工作日 / 周末 / 早晨 / 中午 / 下午 / 晚间
- 展示不同类型商铺的客流分布特征

---

### 模块四：购物流水滚动表（右下）

| 列 | 内容 |
|----|------|
| 购买人 | 头像 + 姓名 |
| 商品名称 | 文字 |
| 价格 | ¥ 金额 |
| 商铺 | 店铺名称 |
| 时间 | HH:mm:ss |

- 每 3 秒自动滚入一条新记录
- 最新记录高亮闪烁

---

### 模块六：品类销售分布（左下） — 饼图/环形图

- 品类：数码电子、服饰鞋包、食品饮料、美妆护肤、家居日用、文体教育
- 环形图中央显示总销售额

## 视觉设计规范

### 配色体系

| 用途 | 色值 | 说明 |
|------|------|------|
| 主背景 | `#0a0e27` | 深蓝黑 |
| 面板背景 | `rgba(6, 30, 65, 0.85)` | 半透明深蓝 |
| 主色调 | `#00d4ff` | 赛博蓝（标题、高亮） |
| 辅助色 | `#00ff88` | 霓虹绿（正向指标） |
| 警告色 | `#ff6b35` | 荧光橙（负向指标） |
| 强调色 | `#a855f7` | 科技紫（图表系列色） |
| 文字主色 | `#e0e6f0` | 亮灰白 |
| 文字次色 | `#7a8ba7` | 暗灰蓝 |

### 视觉特效

1. **边框发光**：面板四周 `box-shadow: 0 0 15px rgba(0, 212, 255, 0.3)`
2. **玻璃拟态**：`backdrop-filter: blur(10px)` + 半透明背景
3. **扫描线动画**：标题区域上方有横向扫描光线
4. **数据流粒子**：背景层流动的微弱光点
5. **装饰元素**：四角装饰线、科技感边框、渐变分隔线

## Open Questions

> [!IMPORTANT]
> **分辨率确认**：方案默认按 **1920×1080** 设计，是否需要适配其他分辨率（如 3840×2160 4K 屏）？

> [!NOTE]
> **后续扩展**：当前方案使用纯前端 Mock 数据。后续如需接入真实后端，只需替换 `mock-data.js` 为 API 调用即可，架构已预留扩展空间。

> [!NOTE]
> **地图交互深度**：当前方案地图支持缩放/平移 + 点击商铺联动。是否需要更多交互功能（如区域框选、热力图叠加等）？
